Tentang Kami

Di era digitalisasi ini, perangkat lunak ibarat jantung dari semua inovasi dan teknologi yang sedang berkembang di penjuru dunia. Jurusan Rekayasa Perangkat Lunak (RPL) di SMK Bina Informatika dapat mempersiapkan siswa untuk menjadi ahli dalam menciptakan solusi teknologi yang membawa perubahan besar terhadap kebutuhan individu, masyarakat, maupun perusahaan. Sekolah Bina Informatika berfokus pada bidang teknologi informasi dan media kreatif, juga memiliki fasilitas modern, kerjasama industri, dan program magang yang relevan. SMK Bina Informatika menciptakan lingkungan belajar yang dinamis dan inspiratif.

Rekayasa Perangkat Lunak

Jurusan Rekayasa Perangkat Lunak (RPL) merupakan salah satu jurusan di SMK Bina Informatika, yang fokus pada pengembangan perangkat lunak dan aplikasi komputer. Siswa jurusan RPL mempelajari berbagai bahasa pemrograman, konsep desain aplikasi, manajemen database, dan teknologi terkini dalam dunia teknologi informasi.

Keunggulan

SMK Bina Informatika memiliki kerjasama dengan berbagai perusahaan teknologi terkemuka yang memberikan peluang magang bagi siswa RPL. Magang ini memberikan pengalaman industri yang berharga dan sering kali menjadi jalur langsung terjun ke dunia kerja setelah lulus.

Kurikulum

Basis Data

Siswa mampu merancang Entity Relationship Diagram (ERD), DFD, PDM serta mengintegrasikan perancangan database tersebut kedalam Software Pengolah Database (DBMS) seperti Ms. Access dan MYSQL menggunakan Query

Pemrograman Website

Siswa mampu membuat aplikasi berbasis website (Framework CI, PHP, HTML, CSS) yang dapat memenuhi kebutuhan industri.

Pemrograman Mobile

Siswa mampu membuat aplikasi perangkat bergerak (Android) yang dapat memenuhi tantangan dan peluang di pasar global

Pemrograman Desktop

Siswa dapat membuat bahan presentasi dengan Software Adobe Flash, Photoshop, Ms.Power Point, Adobe Illustrator, dll.

Design Prototype

Siswa mampu membuat design Prototype Aplikasi sesuai dengan prinsip - prinsip pembuatan aplikasi 8 Golden Rules dan interaksi Manusia dan Komputer (IMK).

Dokumentasi Aplikasi

Siswa mampu membuat dokumentasi aplikasi seperti testing, membuat user manual dan aktifitas pasca pembuatan aplikasi lainnya.

Pemodelan Perangkat Lunak

Siswa mampu menentukan Software Development Life Cycle (SDLC) yang tepat dan mampu menganalisis / mendesign perancangan sebuah aplikasi menggunakan Unified Modeling Language (Activity Diagram, Use Case Diagram, Class Diagram, Sequance Diagram, dll).
